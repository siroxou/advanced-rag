# 1. Record architecture decisions

- **Status:** Accepted
- **Date:** 2026-06-23

## Context

This is a portfolio project meant to demonstrate senior engineering judgment, not just
working code. The *reasoning* behind decisions is as important as the decisions.

## Decision

We keep lightweight [Architecture Decision Records](https://adr.github.io/) in `docs/adr/`.
Each ADR is short: context, decision, consequences. They are numbered and append-only;
superseded ADRs are marked, not deleted.

## Consequences

- Reviewers (and recruiters) can see *why*, not just *what*.
- Trade-offs are explicit and revisitable.
