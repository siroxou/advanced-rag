# Model card: gemma4-rag-lora

A LoRA adapter for Gemma 4 E4B specializing it for grounded, cited
retrieval-augmented answering in this system.

> Status: template. The numbers below are filled in from `app.finetune.evaluate`
> after a training run; the adapter ships only if it beats the base model.

## Intended use

- **Use:** generate answers strictly grounded in provided context, with inline
  `[n]` citations, and refuse faithfully when the context is insufficient.
- **Out of scope:** open-domain answering without retrieved context; any use as a
  standalone knowledge source.

## Training data

- Synthetic, generated by `app.finetune.dataset` from the ingested corpus: one
  grounded question + cited answer per chunk, plus ~25% hard-negative refusals
  (a question paired with a non-answering passage).
- Format: MLX-LM chat JSONL using the production system prompt, so the adapter
  learns the exact citation/refusal contract it is served under.

## Training

- Method: QLoRA via MLX-LM on Apple Silicon (`ml/finetune/lora.yaml`); Unsloth
  QLoRA on a cloud GPU is the equivalent cloud path.
- Base: Gemma 4 E4B (4-bit). Adapter rank 16, ~600 iters. Merge for serving
  (`mlx_lm.fuse`) to avoid the vLLM+adapter quirk, then convert to GGUF for Ollama.

## Limitations and ethics

- Quality is bounded by the teacher model and the corpus; synthetic data can
  inherit teacher biases. The adapter does not change the RBAC/RLS layer - access
  control is enforced in the database regardless of the model.
